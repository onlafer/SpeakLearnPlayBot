# Документация Games API

Это API позволяет интегрировать игры обучающего бота в веб-интерфейсы или мобильные приложения. Логика игр полностью совпадает с Telegram-версией.

## Основная информация

- **Базовый URL:** `http://localhost:8001` (или адрес вашего сервера)
- **Формат данных:** JSON
- **CORS:** Разрешен для всех доменов (`*`)
- **Swagger UI:** `http://localhost:8001/docs` (интерактивное тестирование)

---

## Методы API

### 1. Получение списка игр
`GET /api/games?lang=ru`

Возвращает список доступных игр с названиями на указанном языке.

**Параметры запроса:**
- `lang` (строка, опционально): Код языка (`en`, `ru`, `fr` и т.д.). По умолчанию `en`.

**Ответ (200 OK):**
```json
[
  {
    "game_id": "russian_cases_quiz",
    "name": "Викторина: Падежи"
  },
  {
    "game_id": "translate_word_quiz",
    "name": "Перевод слов"
  }
]
```

---

### 2. Запуск игры
`POST /api/games/{game_id}/start`

Инициализирует новую игровую сессию для пользователя.

**Параметры пути:**
- `game_id`: Идентификатор игры из списка.

**Тело запроса:**
```json
{
  "user_id": 12345,
  "lang": "ru"
}
```
*`user_id` — любой числовой идентификатор пользователя в вашей системе.*

**Ответ (200 OK):**
```json
{
  "game_id": "translate_word_quiz",
  "score": 0,
  "current_question": 1,
  "status": "in_progress",
  "state": {
    "text": "Как переводится слово 'Apple'?",
    "buttons": [
      { "text": "Яблоко", "callback_data": "answer:0:Яблоко" },
      { "text": "Груша", "callback_data": "answer:1:Груша" }
    ]
  }
}
```

---

### 3. Выполнение действия (нажатие кнопки)
`POST /api/sessions/action`

Отправляет выбор пользователя в текущую активную игру.

**Тело запроса:**
```json
{
  "user_id": 12345,
  "callback_data": "answer:0:Яблоко"
}
```
*`callback_data` берется из объекта кнопки, полученного в предыдущем ответе.*

**Ответ (200 OK):**
```json
{
  "game_id": "translate_word_quiz",
  "score": 1,
  "current_question": 2,
  "status": "in_progress",
  "finished": false,
  "state": {
    "text": "Верно! Следующий вопрос: Как переводится 'Banana'?",
    "buttons": [...]
  }
}
```

---

### 4. Получение текущего состояния
`GET /api/sessions/current?user_id=12345`

Позволяет восстановить экран игры при перезагрузке страницы.

**Ответ (200 OK):** Возвращает объект состояния аналогичный `POST /api/games/start`. Если активной игры нет — ошибка 404.

---

### 5. Отмена игры
`POST /api/sessions/cancel`

Завершает текущую сессию пользователя без сохранения прогресса (если применимо).

**Тело запроса:**
```json
{
  "user_id": 12345
}
```

---

## Обработка ошибок

- **404 Not Found:** Сессия не найдена или игра не существует.
- **409 Conflict:** Попытка запустить новую игру, когда старая еще не завершена.
- **500 Internal Server Error:** Ошибка на стороне сервера.

## Особенности реализации

1. **Голосовой ввод:** В текущей версии API игры с голосовыми заданиями (например, произношение) работают только в режиме выбора вариантов (кнопок). Обработка аудиофайлов через API в разработке.
2. **Идентификация:** Поскольку API не требует авторизации, `user_id` должен генерироваться и храниться на стороне клиента (например, в `localStorage`).
3. **Хранение состояния:** Состояние игры хранится в оперативной памяти (SessionManager) и частично в базе данных. При перезапуске сервера активные сессии могут быть сброшены.
